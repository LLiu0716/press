# 与陌生人搭讪的"三次握手"

啥? 三次握手? 这不是一种通信协议么?

三次握手, 作为开发, 必然很熟悉, 或者说, 面试也会被问到, 我就不复述了

那么, 相信了解三次握手的同学都知道, 这个协议带来了什么好处, 如果你不是一个开发者, 不知道这个协议, 请自行百度

# 既然是协议, 可以不遵守吗?

理论上可以, 只需要自己去处理不遵守带来的问题

# 那, 和谈恋爱有毛的关系?

好问题, 哈哈哈!!

安安心心写代码的你, 有没有想过, 去访问一个素不相识的后台, 肯定会, 那么会得到什么结果呢? 这里就要感谢互联网前辈做出的贡献, 哪怕素不相识的服务器, 也不会不理你, 多少也会告诉你一些信息, 根据这些信息, 就知道问题出在哪

# 然鹅, 当你和一个陌生人搭话, 特别是和一个让你心动的异性搭讪

这一切就未知了

当然, 并不是真的未知, 只是被心动的人迷惑了自己

情商高的人, 能根据对方的态度来判断对方返回了什么信息, 是友好? 还是敷衍? 是礼貌, 还是厌恶?

开发都知道, 200 才是我们需要的返回码, 最起码也是 3++ 才可以沟通

她还不了解我, 所以不理我, 我要多表现自己 --- 这是多少情窦初开的少年最纯真的想法

这就是第一次握手失败, 继续尝试, 直到对方有反应, 并且我看到对方反应

这就是三次握手在现实生活中的指导意义

这里注意, 千万别问对方为什么不理自己, 也别问为什么是这样的态度, 对方只会当你丫的有病

我向对方打招呼, 对方做出反应, 我看到了对方的反应, 根据反应选择是否继续沟通

至于对方会怎么回复你, 不是作为搭讪者的我们能控制的, 如果可以控制, 我现在就去访问支付宝的后台往自己的余额前加个 1, 去掉小数点, 然后后面加 N 个 0 岂不美哉 ?

# 需要注意的是

对方不是机器

  • 不需要明确回复你 200, 陌生人的情况下, 只要对方知道了你在和他打招呼, 不表现出讨厌你, 就算三次握手建立成功

  • 在对方没有明显想要和你聊天前, 可以当成互联网里面的 3++ 来处理, 你需要更多的表明你的来意, 表明自己没有恶意

  • 你多次访问一个陌生的服务器, 可能会被拉黑, 人也一样, 当对方开始表现的有些不耐烦, 及时撤走, 实在喜欢对方, 可以等个几天再去偶遇他/她
