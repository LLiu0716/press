# typescript

ts 作为 js 的超集, 在代码规范上起到了很好的补充作用

# 安装 ts

npm i -D typescript

# 生成tsconfig.json

可以直接使用第 3 步的代码

  1. 到package.json添加命令
{
  "scripts": {
    "tsc": "tsc init"
  }
}
  1. 使用命令
npm run tsc
  1. 修改tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "rootDir": "src",
    "target": "ESNext",
    "module": "ESNext",
    "lib": ["ESNext", "DOM", "WebWorker"],
    "outDir": "dist",
    "sourceMap": true,
    "declaration": true,
    "declarationDir": "dist/types",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "moduleResolution": "Node"
  },
  "include": ["src/**/*"]
}

具体配置可以查看官网配置 tsconfig.json 项目配置open in new window

这时可以创建一个 ts 文件到src文件夹下 , 试试 typescript 是否生效

src/index.ts

const main = (a: number, b: number): number => {
  return a - b
}
  • 添加命令
{
  "scripts": {
    "dev": "tsc --watch"
  }
}

  • 查看 dist 文件夹

  • 得到一个 index.js 文件

  • 得到一个 index.js.map 文件

  • 得到一个 types/index.d.ts 文件

# 配置方法

可以直接使用第 2 步的代码

  1. 安装eslint后通过eslint内置命令生成一份配置文件, 或者直接写一个eslint认识的文件, 具体格式请百度 这里选择了es 模块open in new window, 所以使用.cjs文件

差异请百度: es 和 commonjs 的差异

  • 使用命令

    npm i eslint -D
npx eslint --init

  • 根据提示一步步选择

  1. 修改自动生成/自己创建的.eslintrc.cjs文件
module.exports = {
  parser: '@typescript-eslint/parser',
  extends: ['standard', 'eslint-config-standard-with-typescript'],
  parserOptions: {
    ecmaVersion: 'latest',
    sourceType: 'module',
    project: './tsconfig.json'
  },
  plugins: ['@typescript-eslint'],
  rules: {}
}
  1. 这时可以试试 eslint 是否生效
  • 添加命令
{
  "scripts": {
    "lint": "eslint src --ext .js,.ts --cache --fix"
  }
}
  • 运行命令
npm run lint

得到 eslint 报错 'main' is assigned a value but never used

# prettier

包名当前使用版本
prettier8.22.0
eslint-config-prettier8.5.0
eslint-plugin-prettier4.2.1

代码格式化工具

能快速保存并格式化代码, 作用和eslint有部分重叠

  1. 安装包
npm i -D prettier eslint-config-prettier eslint-plugin-prettier
  1. 添加配置文件.prettierrc.json
{
  "semi": false,
  "singleQuote": true,
  "trailingComma": "none"
}
  1. 修改 .eslintrc.cjs文件
module.exports = {
  parser: '@typescript-eslint/parser',
- extends: ['standard', 'eslint-config-standard-with-typescript'],
+ extends: ['standard', 'eslint-config-standard-with-typescript', 'prettier'],
  parserOptions: {
    ecmaVersion: 'latest',
    sourceType: 'module',
    project: './tsconfig.json'
  },
- plugins: ['@typescript-eslint'],
+ plugins: ['@typescript-eslint', 'prettier'],
- rules: {},
+ rules: {
    'prettier/prettier': 2
  }
}
  1. 代码中默认格式化工具选择 prettier 并格式化代码

# husky

包名当前使用版本
husky8.0.1
lint-staged13.0.3

安装包

npm i husky lint-staged -D
npx husky install

  • 自动得到一个 .husky 文件

  • 在 .husky 文件夹下创建 pre-commit 和 commit-msg 钩子文件

.husky/commit-msg

npx --no -- commitlint --edit $1

.husky/pre-commit

npx --no -- lint-staged
  • 在package.json添加配置文件
{
  "lint-staged": {
    "*.{js,ts}": ["prettier --write", "eslint --fix"]
  }
}

这样提交就会自动格式化代码, 还可以加入其他命令, 例如知道测试命令

lint-staged只对修改的文件起作用

统一代码提交格式

包名当前使用版本
@commitlint/cli17.0.3
@commitlint/config-conventional17.0.3
commitizen4.2.5

安装包

npm i -D @commitlint/cli @commitlint/config-conventional commitizen

# 这里会有 2 套配置 , 推荐使用 Angular 团队规范的 preset

  • 添加配置文件 commitlint.config.cjs 到根文件夹
module.exports = {
  extends: ['@commitlint/config-conventional', 'cz'],
  rules: {
    'subject-case': [0, 'never']
  }
}

# Angular 团队规范的 preset 配置方法

包名当前使用版本
cz-conventional-changelog3.3.0

纯英文, 对英语不好的同学不太友好

  1. 安装包
npm i -D cz-conventional-changelog
  1. 添加配置到package.json文件
{
  "scripts": {
    "lint": "eslint src --ext .js,.ts --cache --fix",
    "commit": "git cz"
  },
  "config": {
    "commitizen": {
      "path": "node_modules/cz-conventional-changelog"
    }
  }
}
  1. 使用命令并填写相对应的参数, 如同邮件般填写 commit
npm run commit

# 自定义 preset 配置方法

这里使用 cz-gitopen in new window

包名当前使用版本
cz-git1.3.10
cz-cli1.3.10
conventional-changelog-lint-config-cz0.3.0
commitlint-config-cz0.13.3
  1. 安装包
npm i -D cz-git cz-cli conventional-changelog-lint-config-cz commitlint-config-cz
  1. 添加配置文件 cz.config.cjs 到根文件夹
'use strict'
module.exports = {
  types: [
    { value: 'feat', name: 'feat:     新增功能 | A new feature' },
    { value: 'fix', name: 'fix:      修复缺陷 | A bug fix' },
    { value: 'docs', name: 'docs:     文档更新 | Documentation only changes' },
    {
      value: 'style',
      name: 'style:    代码格式 | Changes that do not affect the meaning of the code'
    },
    {
      value: 'refactor',
      name: 'refactor: 代码重构 | A code change that neither fixes a bug nor adds a feature'
    },
    {
      value: 'perf',
      name: 'perf:     性能提升 | A code change that improves performance'
    },
    {
      value: 'test',
      name: 'test:     测试相关 | Adding missing tests or correcting existing tests'
    },
    {
      value: 'ci',
      name: 'ci:       持续集成 | Changes to our CI configuration files and scripts'
    },
    { value: 'revert', name: 'revert:   回退代码 | Revert to a commit' },
    {
      value: 'chore',
      name: 'chore:    其他修改 | Other changes that do not modify src or test files'
    }
  ],

  messages: {
    type: '选择你要提交的类型 :',
    scope: '选择一个提交范围（可选）:',
    customScope: '请输入自定义的提交范围 :',
    subject: '填写简短精炼的变更描述 :\n',
    body: '填写更加详细的变更描述（可选）。使用 "|" 换行 :\n',
    breaking: '列举非兼容性重大的变更（可选）。使用 "|" 换行 :\n',
    footer: '列举关联issue (可选) 例如: #31, #I3244 :\n',
    confirmCommit: '是否提交或修改commit ? (Y/N)'
  },

  allowCustomScopes: false,
  allowCustomIssuePrefixs: false,
  allowEmptyIssuePrefixs: false,
  subjectLimit: 100,
  emptyScopesAlias: '跳过',
  defaultBody: '',
  defaultScope: '',
  defaultSubject: '',
  customScopesAlias: '以上都不是？我要自定义',
  allowBreakingChanges: ['feat', 'fix']
}

更多配置选项请查看相关配置open in new window

  1. 添加配置到package.json文件
{
  "scripts": {
    "lint": "eslint src --ext .js,.ts --cache --fix",
    "commit": "git cz"
  },
  "config": {
    "commitizen": {
      "path": "node_modules/cz-git",
      "czConfig": "cz.config.cjs"
    }
  }
}
  1. 使用命令并填写相对应的参数, 如同邮件般填写 commit
npm run commit
